DESIGN PRINCIPLES
Six principles behind every decision
Not guidelines — hard constraints that shaped the schema from day one.
01 — REAL-TIME
Real-Time by Default
CDC → Kafka → BigQuery streaming. Batch is a fallback, not the baseline.
02 — IDENTITY
Identity-First Data Model
Every transaction links to a canonical party record before analytics run.
03 — SIMPLICITY
Radical Simplicity = Performance
FX, IBAN, and IR audit fields are first-class columns — not JSON keys.
04 — CLOUD
Cloud-Native Economics
Storage/compute separation. Pay for queries, not idle servers.
05 — APIs
APIs & Pipelines
Every layer exposes a clean, independently deployable contract.
06 — AI/ML
AI / ML Intelligence First
Structured columns and tokenized PII are purpose-built for feature engineering.
