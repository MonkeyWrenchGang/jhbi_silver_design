ACT 03
Cross-rail insights only the identity graph unlocks
Without resolved party IDs, these queries require expensive joins across rail-specific tables and still produce ambiguous results.
8→1
Total $ flow per party, all rails
Sum inflow and outflow for a single entity across ACH, Wire, Zelle, RTP — one WHERE originator_party_id = ?.
SELECT SUM(settlement_amount)
FROM unified_transactions
WHERE originator_party_id = 'P-4821'
AND status = 'SETTLED'
⇄
Counterparty exposure across rails
Find every rail where Party A sent money to Party B — even if they used different account numbers each time.
SELECT payment_type,
SUM(settlement_amount)
FROM unified_transactions
WHERE originator_party_id = 'P-4821'
AND beneficiary_party_id = 'P-0093'
GROUP BY 1
UBO
Ultimate beneficial owner roll-up
Aggregate all transaction volume across entities owned by the same UBO using the party_relationship table.
SELECT SUM(t.settlement_amount)
FROM unified_transactions t
JOIN party_relationship r
ON t.originator_party_id = r.child_party_id
WHERE r.parent_party_id = 'UBO-112'
AND r.relationship_type = 'UBO'
JT
Joint account cross-rail activity
Identify all payment activity tied to a joint account, regardless of which account holder initiated the transaction or which rail was used.
SELECT p.party_id, t.payment_type,
SUM(t.settlement_amount)
FROM unified_transactions t
JOIN party_account_link l
ON t.originator_party_id = l.party_id
WHERE l.account_id = 'ACCT-7734'
GROUP BY 1,2
IR
Identity resolution quality by rail
Track what % of transactions per rail have resolved party IDs — surfacing data quality gaps at the source system level.
SELECT payment_type,
COUNTIF(originator_party_id IS NOT NULL)
/ COUNT(*) AS ir_rate
FROM unified_transactions
WHERE DATE(transaction_ts) >= DATE_SUB(CURRENT_DATE,INTERVAL 30 DAY)
GROUP BY 1
ON-US
On-us detection without a lookup
Because both parties are resolved at ETL time, is_on_us is a derived boolean on every row — zero runtime joins in query paths.
SELECT payment_type,
COUNTIF(is_on_us) AS on_us_count,
SUM(CASE WHEN is_on_us
THEN settlement_amount END) AS on_us_vol
FROM unified_transactions
GROUP BY 1
